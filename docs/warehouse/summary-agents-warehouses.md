# Сводка (warehouse/summaries): агенты и несколько складов

Доработка раздела «Сводка» (`/crm/warehouse/documents/sale?tab=summary`).
Фронт уже обновлён (июль 2026) и шлёт новые поля — бэку нужно их поддержать.

## 1. Склады: выбранные или все

Сейчас сводка жёстко привязана к одному складу (`warehouse`, обязательное).
Нужно: сводка по **нескольким складам** или по **всем складам компании**.

### Контракт

`POST /warehouse/summaries/` — новые поля:

```json
{
  "name": "Общая",
  "date": "2026-07-15",
  "type": "general",
  "agents": [],

  "all_warehouses": true,      // ← новое: сводка по всем складам компании
  "warehouses": [],            // ← новое: список uuid складов (когда all_warehouses=false)

  "warehouse": "uuid"          // legacy: фронт продолжает слать для совместимости
}
```

Правила:

- `all_warehouses=true` → `warehouses` игнорируется, в сводку попадают накладные **всех** складов компании за дату.
- `all_warehouses=false` → `warehouses` обязателен и непуст; накладные фильтруются по этим складам.
- Легаси `warehouse` (одиночный uuid) принимать как `warehouses=[warehouse]`, если новые поля не переданы — старые клиенты не ломаются.
- `PATCH /warehouse/summaries/{id}/` — разрешить менять `all_warehouses`/`warehouses` (с пересборкой снапшота, как при `regenerate`).

### Ответ (list/detail)

```json
{
  "all_warehouses": false,
  "warehouses": [{ "id": "uuid", "name": "Основной склад" }],
  "warehouse": { "id": "uuid", "name": "Основной склад" }   // legacy: первый из списка
}
```

## 2. Агенты: любой сотрудник, кроме admin/owner

На фронте список «Агенты» в сводке теперь совпадает со списком «Агент» при
создании продажи: **все сотрудники компании, кроме ролей admin/owner** (раньше
фронт фильтровал по `role == "agent"` и список оказывался пустым).

Просьба к бэку: в `agents` сводки принимать id **любого сотрудника компании**
(без валидации «роль строго agent»), иначе выбор, который теперь показывает
фронт, будет отклоняться. Фильтр списка `?agent=` — аналогично.

## 3. Что уже работает и не меняется

- `documents[]` в detail-ответе содержат `id` накладных — фронт по ним
  подгружает полные документы (`GET /warehouse/documents/{id}/`) и печатает
  каждую накладную внутри PDF сводки в формате продажи, в двух экземплярах.
  Бэку тут ничего делать не нужно — только не убирать `id` из `documents[]`.
- `type` (`general` | `by_agents`), календарь, фильтры списка — без изменений.
