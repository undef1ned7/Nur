# Сводка по контрагентам за период (Сальдо / Оборот)

Для страницы `/crm/warehouse/counterparties`, вкладки **Клиент** и **Поставщик**.
Над списком показываются три блока показателей за выбранный период:
**Сальдо на начало периода**, **Оборот за период**, **Сальдо на конец периода** — каждый с
разбивкой на **Дебет** и **Кредит**. Фронт уже готов и вызовет эндпоинт ниже.

## Период (с фронта)
По умолчанию — **текущий месяц** (с 1-го до последнего числа). Пользователь выбирает период:
по **месяцу**, по **году** или **кастомным** диапазоном. На бэкенд всегда уходят готовые
`date_from` и `date_to` (`YYYY-MM-DD`, включительно).

## Эндпоинт — `GET /api/warehouse/counterparties/balance-summary/` (NEW)

**Query-параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `type` | `client` \| `supplier` | Тип контрагентов (соответствует активной вкладке) |
| `date_from` | date `YYYY-MM-DD` | Начало периода (включительно) |
| `date_to` | date `YYYY-MM-DD` | Конец периода (включительно) |

Все данные ограничены компанией пользователя (мультиарендность), как в остальном warehouse API.

**Ответ `200`:**
```jsonc
{
  "opening":  { "debit": 12000.00, "credit": 3000.00 },  // Сальдо на начало (на 00:00 date_from)
  "turnover": { "debit": 45000.00, "credit": 41000.00 }, // Обороты внутри [date_from, date_to]
  "closing":  { "debit": 17000.00, "credit": 1000.00 }   // Сальдо на конец (на 23:59 date_to)
}
```

### Смысл показателей
- **opening (Сальдо на начало периода)** — накопленные дебет и кредит по всем операциям
  контрагентов данного типа **строго до** `date_from`.
- **turnover (Оборот за период)** — суммы дебета и кредита по операциям **внутри** диапазона
  `[date_from, date_to]` включительно.
- **closing (Сальдо на конец периода)** — состояние **на конец** `date_to`, то есть
  `opening + turnover` по каждой из сторон (`closing.debit = opening.debit + turnover.debit`,
  аналогично для `credit`).

Значения — агрегаты по **всем** контрагентам выбранного типа (сумма по компании), в валюте учёта.
Числа — с двумя знаками после запятой; при отсутствии данных возвращать нули, а не `null`.

### Что считать дебетом и кредитом
- **Дебет** — задолженность контрагента перед нами (отгрузки/продажи в долг, приход долга).
- **Кредит** — наша задолженность перед контрагентом / поступления от него (оплаты, авансы,
  возвраты).

Источник — те же операции, что формируют долги и движение по контрагенту
(см. `getCounterpartyDebts`, `getCounterpartyMoneyOperations`, акт сверки). Логика дебет/кредит
должна совпадать с актом сверки (`reconciliation/json`), чтобы цифры сходились.

## Производительность
- Один агрегирующий запрос: `opening` — `SUM` по операциям с `date < date_from`;
  `turnover` — `SUM` с `date_from <= date <= date_to`, сгруппировано по стороне (дебет/кредит).
- Индекс по `(company, counterparty_type, date)`.

## Чек-лист (сводка)
- [ ] Эндпоинт `GET warehouse/counterparties/balance-summary/`.
- [ ] Параметры `type`, `date_from`, `date_to`; ограничение по компании.
- [ ] `opening` (до `date_from`), `turnover` (в периоде), `closing = opening + turnover`.
- [ ] Дебет/кредит считаются так же, как в акте сверки.
- [ ] Пустые периоды → нули, ответ `200` (не `404`/`null`).

# Часть 2. Те же показатели по КАЖДОЙ строке списка контрагентов

В таблице/карточках списка контрагентов каждая строка показывает те же 6 колонок
(Сальдо на начало Дт/Кт, Оборот Дт/Кт, Сальдо на конец Дт/Кт) — но **по конкретному
контрагенту** за выбранный период. Итоговая строка «Итого» суммирует их и должна сходиться
со сводкой из Части 1.

Фронт **уже готов**: список грузится через `GET warehouse/.../counterparties` (`listCounterparties`),
и при выбранном периоде делается второй запрос с `period_start` / `period_end`, после чего
`analytics` каждого контрагента подмешивается в строку.

## Эндпоинт списка — `GET` counterparties (существующий)
Нужно, чтобы он принимал период и возвращал по каждому контрагенту блок `analytics`:

| Параметр | Описание |
|----------|----------|
| `type` | `client` \| `supplier` \| `both` (как сейчас) |
| `period_start` | дата `YYYY-MM-DD`, начало периода (включительно) — **NEW** |
| `period_end` | дата `YYYY-MM-DD`, конец периода (включительно) — **NEW** |

> Имена параметров списка — `period_start`/`period_end` (так уже шлёт фронт). В сводке из
> Части 1 — `date_from`/`date_to`. Это разные эндпоинты; менять не нужно, просто учесть.

При переданных `period_start`/`period_end` допустимо вернуть **только** контрагентов с
движением за период (фронт подмешает их analytics в полный список по `id`).

## Поля `analytics` у элемента списка
```jsonc
{
  "id": "…", "name": "…", "type": "client", "agent": null,
  "analytics": {
    "debts": {
      "opening_debit": 12000.00,   // Сальдо на начало — дебет (до period_start)
      "opening_credit": 0.00,      // Сальдо на начало — кредит
      "turnover_debit": 45000.00,  // Оборот за период — дебет
      "turnover_credit": 41000.00, // Оборот за период — кредит
      "closing_debit": 17000.00,   // Сальдо на конец — дебет (= opening + turnover)
      "closing_credit": 0.00       // Сальдо на конец — кредит
    }
  }
}
```

### Совместимость с текущим фронтом (важно)
Фронт читает поля в таком порядке приоритета (первое непустое):
- **opening**: `opening_debit` / `opening_credit` (либо `start_debit`/`period_start_debit`).
- **turnover**: `turnover_debit` / `turnover_credit` (либо `period_debit`, либо `cash.received`/`cash.paid`).
- **closing**: `closing_debit` / `closing_credit` → `counterparty_owes_company` /
  `company_owes_counterparty` → разбор знака `debts.balance`.

Рекомендуется отдавать **явные** `opening_*`, `turnover_*`, `closing_*` — это самый
надёжный и однозначный вариант.

### Бизнес-логика (одинаково для строки и для сводки)
- **Дебет** — задолженность контрагента перед компанией.
- **Кредит** — задолженность компании перед контрагентом.
- `closing_debit = opening_debit + turnover_debit`, `closing_credit = opening_credit + turnover_credit`.
- Те же правила, что в акте сверки (`reconciliation/json`) — цифры строк, «Итого» и сводки Части 1
  должны совпадать.

## Чек-лист (строки списка)
- [ ] Список принимает `period_start`/`period_end`.
- [ ] У каждого контрагента `analytics.debts` с `opening_*`, `turnover_*`, `closing_*` за период.
- [ ] Без периода — поведение как сейчас (текущее сальдо).
- [ ] Сумма строк = сводка из Части 1 за тот же период и тип.
