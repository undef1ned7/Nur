# Покупка и приход в складских документах

Краткий разбор отличий типов документов **Покупка** (PURCHASE) и **Приход** (RECEIPT) на основе кода в `Documents.jsx` и `CreateSaleDocument.jsx`.

---

## 1. Типы документов

- **Покупка** → `doc_type: "PURCHASE"`
- **Приход** → `doc_type: "RECEIPT"`

В списке документов они различаются только значением `doc_type` в запросе к API.

---

## 2. Отличия в CreateSaleDocument.jsx

### Контрагент (поставщик)

| | Покупка (PURCHASE) | Приход (RECEIPT) |
|---|---|---|
| Обязательность | Контрагент **обязателен** | Контрагент **не обязателен** |
| Тип контрагента | Только **поставщики** (SUPPLIER или BOTH) | Не используется |

Для прихода поле контрагента в форме не требуется.

### Цена по умолчанию

- **Покупка** — в позиции подставляется **закупочная цена**: `product.purchase_price ?? product.price`.
- **Приход** — подставляется **цена продажи**: `product.price`.

### Оплата (сразу / в долг)

- **Покупка** — показывается выбор **payment_kind** («Сразу» / «В долг»), значение уходит в API.
- **Приход** — поле оплаты **не показывается** и не передаётся.

### API создания

- **Покупка** → `warehouseAPI.createPurchaseDocument(payload)`
- **Приход** → `warehouseAPI.createReceiptDocument(payload)`

---

## 3. Сводная таблица

| Критерий | Покупка (PURCHASE) | Приход (RECEIPT) |
|----------|--------------------|------------------|
| Контрагент | Обязателен (поставщик) | Не обязателен |
| Цена по умолчанию | Закупочная (`purchase_price`) | Цена продажи (`price`) |
| Оплата (сразу/в долг) | Есть (`payment_kind`) | Нет |
| Эндпоинт API | `createPurchaseDocument` | `createReceiptDocument` |

---

## 4. Смысл операций

- **Покупка** — документ закупки **у поставщика**: кто поставил, по какой цене, оплата сразу или в долг.
- **Приход** — оприходование товара **на склад без привязки к поставщику** (ручной приход, найденный товар и т.п.); контрагент и оплата в форме не используются.
