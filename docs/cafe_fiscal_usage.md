# Фискальная касса в кафе — инструкция

Пошаговое руководство: как настроить и пользоваться Fiscal Connector в модуле **Кафе** Nur.

> Техническая документация по API и архитектуре: [`cafe_fiscal_connector.md`](./cafe_fiscal_connector.md)

---

## 1. Что это и зачем

**Fiscal Connector** — программа на кассовом компьютере, которая пробивает **юридически значимые чеки** через SAM-карту и фискальный модуль (ФМ).

| Слой | За что отвечает |
|------|-----------------|
| **Nur (браузер)** | Заказы, столы, оплата, учёт, офлайн-синк |
| **Fiscal Connector** (`localhost:8080`) | Фискальная смена, чеки, внесения, Z-отчёт |
| **Nur backend** (`/cafe/fiscal/...`) | Хранение настроек кассы, ФД/ФМ, журнал чеков |

Если фискализация **выключена** (`enabled: false`) — кафе работает как раньше: свой ESC/POS-чек без Fiscal Connector.

---

## 2. Что нужно на кассовом ПК

### 2.1. Оборудование и ПО

1. **SAM-карта** — вставлена в кардридер.
2. **Fiscal Connector** — установлен через `FiscalConnectorSetup.exe` и **запущен** (слушает `http://localhost:8080`).
3. **Браузер** с открытым Nur (лучше тот же ПК, где стоит коннектор).
4. **Интернет** — нужен **только утром** для открытия смены. После этого чеки можно бить без сети.

### 2.2. Данные, которые понадобятся один раз

| Поле | Где взять |
|------|-----------|
| РНМ кассы | 16 символов, с регистрации ККМ |
| PIN SAM-карты | 5 цифр |
| Логин / пароль | Учётная запись в Fiscal Connector |
| URL коннектора | Обычно `http://localhost:8080` |

## 3. Первоначальная настройка (администратор)

Настройки хранятся на backend: `GET/PATCH /cafe/fiscal/settings/`.  
Пока отдельного экрана в интерфейсе может не быть — первый раз настройку делает администратор через API или админку.

### 3.1. Включить фискализацию

Пример запроса (с JWT пользователя Nur):

```http
PATCH /cafe/fiscal/settings/
Content-Type: application/json

{
  "enabled": true,
  "connector_base_url": "http://localhost:8080",
  "registration_number": "0123456789012345",
  "pin": "12345",
  "login": "cafe@example.com",
  "password": "ваш_пароль",
  "default_vat_code": 12,
  "default_st_code": 0,
  "default_calc_item_attr_code": 1,
  "default_measure": "шт",
  "receipt_width": 384
}
```

| Поле | Описание |
|------|----------|
| `enabled` | `true` — включить фiscal; `false` — работа как раньше |
| `connector_base_url` | Адрес Fiscal Connector |
| `registration_number` | РНМ (16 символов) |
| `pin` | PIN SAM-карты |
| `login`, `password` | Авторизация в коннекторе |
| `default_vat_code` | НДС по умолчанию, если у блюда не задан |
| `default_st_code` | НСП по умолчанию |
| `default_calc_item_attr_code` | Признак предмета расчёта по умолчанию |
| `default_measure` | Единица измерения по умолчанию (`шт`) |
| `receipt_width` | Ширина чека в пикселях (обычно `384`) |

Проверка:

```http
GET /cafe/fiscal/settings/
```

В ответе должно быть `"enabled": true` и заполнены РНМ, PIN, логин.

### 3.2. Налоговые коды у блюд

В **Меню → карточка блюда** раскройте блок **«Фискальные коды (ккм)»** и при необходимости укажите:

| Поле | Смысл |
|------|-------|
| Код НДС (vat) | Например `12` для VAT_12, `0` для VAT_0 |
| Код НСП (st) | 0–5 |
| Признак предмета расчёта | Обычно `1` |
| Единица измерения | `шт`, `порция` и т.д. |
| ТНВЭД (sgtin) | Необязательно |

Если поля пустые — backend подставит `default_*` из настроек кассы.

**Рекомендация:** один раз при открытой смене сверьте коды с коннектором (`GET /driver/cash-register/available-tax-rates`) и проставьте их у основных позиций меню.

### 3.3. Принтер (как раньше)

Кухонные и дополнительные чеки по-прежнему идут через **Настройки → Печать** (`cafe_receipt_printer`, USB/Wi‑Fi, printer-bridge).

Фiscal Connector отдаёт **фiscalный** чек (PDF/JSON с ФД). Печать фiscalного чека зависит от настроек коннектора и принтера на кассовом ПК.

## 4. Рабочий день кассира

### 4.1. Утро — открытие смены

1. Убедитесь, что **FiscalConnectorSetup.exe запущен** и SAM-карта вставлена.
2. Проверьте **интернет**.
3. В Nur: **Кафе → Заказы** → если смены нет, откроется экран **«Открытие смены»**.
4. Введите **начальную сумму наличных** в кассе.
5. Нажмите **«Открыть смену»**.

Если фискализация включена, система автоматически выполнит:

```
Верификация SAM-карты (verify-pin)
    ↓
Авторизация в коннекторе (auth, токен 5 мин)
    ↓
Проверка состояния смены
    ↓
Открытие фiscalной смены (open-shift)
    ↓
Фиксация смены в Nur (/cafe/fiscal/shift/open/)
    ↓
Обычное открытие смены в Nur + offline snapshot
```

На экране будут шаги вида «⏳ Верификация SAM-карты…», «⏳ Авторизация…» и т.д.

**Важно:** не закрывайте окно и не отвлекайтесь между авторизацией и открытием — токен живёт **5 минут**.

При успехе — сообщение **«Смена успешно открыта»**.

### 4.2. День — заказы и оплата

Работа с заказами **не меняется**: стол, позиции, кухня, официант — как обычно.

При нажатии **«Оплатить»**:

1. Выберите способ оплаты (наличные, карта, split, долг и т.д.).
2. Подтвердите оплату.

Если `enabled: true`, Nur **сам**:

1. Запросит у backend готовое тело чека:  
   `GET /cafe/fiscal/orders/{id}/receipt-payload/`
2. Пробьёт чек на коннекторе:  
   `POST localhost:8080/driver/cash-register/receipt`
3. Сохранит ФД/ФМ в Nur (в фоне):  
   `POST /cafe/fiscal/orders/{id}/receipt/`
4. Проведёт оплату заказа:  
   `POST /cafe/orders/{id}/pay/`

Кассиру **не нужно** вручную собирать позиции чека — это делает backend.

Если фiscalный чек **не пробился**, появится предупреждение («Фiscalный чек не пробит: …»). Оплата в Nur при этом **может пройти** — уточните у администратора политику на вашей точке (пробивать чек до выдачи гостю).

### 4.3. Офлайн (нет интернета)

После **успешного открытия смены утром** (при интернете):

| Действие | Интернет |
|----------|----------|
| Пробить фiscalный чек | **Не нужен** |
| Закрыть заказ локально | Не нужен (очередь синка) |
| Синхронизировать заказы на сервер | Нужен, когда сеть появится |

При оплате без сети Nur:

- пробивает чек через коннектор (локально);
- кладёт заказ в офлайн-очередь;
- освобождает стол.

Когда интернет вернётся — синхронизация идёт через обычный механизм офлайна (`useCafeSync`).

**Без интернета утром открыть смену нельзя** — нужны `auth` и `open-shift` на коннекторе.

### 4.4. Вечер — закрытие смены

Закрытие фiscalной смены на коннекторе (`close-shift`) и запись в Nur (`POST /cafe/fiscal/shift/close/`) **планируются отдельным шагом** в интерфейсе.

Пока закрывайте смену в Nur как раньше. Перед закрытием на коннекторе (если делаете вручную):

1. Проверьте наличные: `GET /driver/cash-transaction`
2. При необходимости изымите: `POST /driver/cash-transaction/withdraw`
3. Закройте смену: `POST /driver/close-shift`

Если в кассе остались деньги, коннектор вернёт ошибку **40919** — «Изымите наличные перед закрытием».

## 5. Типовые ситуации

### SAM-карту вытащили и вставили снова

Нужна повторная верификация PIN. Закройте и снова откройте смену или перезапустите Fiscal Connector и повторите `verify-pin` (при следующем открытии смены это сделает Nur автоматически).

### «Fiscal Connector недоступен»

- Запущен ли `FiscalConnectorSetup.exe`?
- Открывается ли в браузере `http://localhost:8080`?
- Nur и коннектор на **одном** компьютере?

### «Смена не открыта» (40918)

Фiscalная смена не открыта на коннекторе. Откройте смену через Nur (утром, с интернетом).

### «Требуется повторная авторизация» (4011)

Токен истёк (5 минут) или смена была закрыта. Откройте смену заново — Nur выполнит `auth` снова.

### «Смена открыта более 24 ч» (40920)

Закройте фiscalную смену и откройте новую.

### «Неверный PIN» (4008)

Проверьте PIN в настройках (`PATCH /cafe/fiscal/settings/`, поле `pin`).

### «SAM-карта не выбрана» (40416)

В Fiscal Connector выберите SAM-карту в интерфейсе программы.

### CORS / запросы к localhost блокируются

Браузер может блокировать вызовы на `localhost:8080` с другого домена. Решения:

- открывать Nur на том же ПК, где коннектор;
- использовать Electron/Tauri или локальный proxy (например printer-bridge на `127.0.0.1`).

## 6. Что видит администратор

### Журнал фiscalных документов

```http
GET /cafe/fiscal/receipts/?kind=sale&order={uuid_заказа}
```

Фильтры: `kind` (`sale`, `return`, `deposit`, `withdraw`, …), `order`, `?branch=`.

### Состояние смены в Nur

```http
GET /cafe/fiscal/shift/state/
```

Для сверки с коннектором; источник правды по факту — `GET /driver/state-shift`.

### Журнал смен

```http
GET /cafe/fiscal/shifts/
```

## 7. Возврат по заказу

1. Найти исходный чек продажи:  
   `GET /cafe/fiscal/receipts/?order={id}&kind=sale`
2. Взять `fd_number` и `fn_serial_number`.
3. Получить payload возврата:  
   `GET /cafe/fiscal/orders/{id}/receipt-payload/?operation_type=INCOME_RETURN&origin_fd_number=...&origin_fn_serial_number=...`
4. Пробить на коннекторе и записать:  
   `POST /cafe/fiscal/orders/{id}/receipt/` с `"kind": "return"`.

UI для возврата может быть добавлен позже; логика на backend уже предусмотрена.

## 8. Краткая шпаргалка

```
ПЕРВЫЙ РАЗ (админ)
  PATCH /cafe/fiscal/settings/  → enabled=true, РНМ, PIN, login, password
  Меню → блюда → фiscalные коды (или дефолты в settings)

КАЖДОЕ УТРО (кассир)
  1. Запустить Fiscal Connector + SAM в ридере
  2. Интернет включён
  3. Кафе → Открыть смену → сумма → «Открыть смену»

ОПЛАТА
  Оплатить заказ как обычно → чек бьётся автоматически (если enabled)

ОФЛАЙН
  После утреннего open-shift → чеки без интернета OK, синк позже

ВЫКЛЮЧИТЬ ФИСКАЛ
  PATCH /cafe/fiscal/settings/  →  "enabled": false
  → снова только ESC/POS, без коннектора
```
