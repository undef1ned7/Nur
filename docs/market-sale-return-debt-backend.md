# Магазин: возврат продажи должен корректировать ДОЛГ клиента (частичный и полный)

Касается кассы Магазина (`/crm/market/cashier`) и возврата продажи:
`POST /main/pos/sales/{saleId}/return/` (для агента — `/main/agents/me/sales/{saleId}/return/`).

Связанная задача по кассе: см. `docs/market-sale-return-expected-cash-backend.md`.

## Проблема
При возврате продажи, оформленной **в долг**, задолженность клиента не корректируется.

## Требование
Если продажа была **в долг** (полностью или частично — есть непогашенный остаток по клиенту),
при возврате нужно **уменьшать задолженность клиента** на сумму возвращённых позиций.

- **Полный возврат**: списывается вся связанная с продажей задолженность (долг по этой продаже
  обнуляется).
- **Частичный возврат**: долг уменьшается на **сумму возвращённых позиций** (пропорционально
  фактически возвращённому), а не на всю продажу.
- Пересчитывать корректно, если по продаже уже была **частичная оплата**: сначала возврат
  уменьшает непогашенный долг; если возвращаемая сумма превышает остаток долга — разницу вернуть
  деньгами (нал/безнал по способу оплаты), долг не уходит в минус.
- Обновлять **баланс клиента / карточку долга** и историю операций по контрагенту, чтобы цифры
  сходились с актом сверки и списком долгов.
- Возврат долговой продажи **не трогает `expected_cash`** (наличные не двигались), но должен
  отразиться в задолженности.

## Пример (частичный)
Продажа в долг на 1000, оплачено 0. Возврат позиций на 300 → долг клиента: 1000 → **700**.

## Чек-лист
- [ ] Полный возврат долговой продажи обнуляет связанный долг клиента.
- [ ] Частичный возврат уменьшает долг на сумму возвращённых позиций.
- [ ] Учтена частичная предоплата: возврат сверх остатка долга возвращается деньгами, долг не уходит в минус.
- [ ] Обновлены баланс клиента, карточка долга и история операций (сходятся с актом сверки).
- [ ] Долговой возврат не изменяет `expected_cash`.
