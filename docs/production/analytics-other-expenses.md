# Производство — Аналитика: прочие расходы и чистая прибыль

**Страница:** `/crm/production/analytics` (фронт: `src/Components/Sectors/Production/Analytics/ProductionAnalytics.jsx`).
**Эндпоинт:** `GET /main/owners/analytics/` (сводка аналитики производства, блок `summary`).
**Статус:** фронт готов — карточки «Прочие расходы» и «Чистая прибыль» появятся автоматически, как только бэкенд начнёт отдавать новые поля. Правки нужны только на бэкенде.

---

## 1. Проблема (жалоба клиента)

В аналитике при расчёте прибыли учитывается только себестоимость проданного. «Прочие расходы» (аренда, коммуналка, транспорт, хознужды и другие расходы из кассы) нигде не вычитаются, поэтому владелец видит завышенную прибыль и не может оценить реальный результат.

## 2. Требуемое поведение

Существующие показатели **не менять** (валовая прибыль остаётся «выручка − себестоимость» — это корректная валовая метрика). Добавить в `summary` три новых поля:

| Поле | Формула |
|---|---|
| `other_expenses_total` | Σ расходных операций кассы за период, **кроме** закупок сырья/товаров |
| `net_profit` | `gross_profit − other_expenses_total` |
| `net_margin_percent` | при `revenue > 0`: `net_profit / revenue × 100`, иначе `0` |

### Что входит в «прочие расходы»

1. Берутся операции кассы `type=expense` со статусом `approved` за выбранный период (те же фильтры периода/филиала, что и у остальной сводки).
2. **Исключаются** операции закупки сырья и товаров — они уже сидят в себестоимости, иначе будет двойной вычет. Признак исключения: операции, созданные потоками закупки (название «Закупка товара: …» / привязка к закупке), и любые операции с `source_business_operation_id`, указывающим на закупку.
3. Зарплатные выплаты, аренда, транспорт и прочее ручное — **входят**.
4. Операции со статусом `pending`/`rejected` — не входят.

## 3. Контракт ответа

К существующему `summary` добавляются поля (Decimal-строки, как остальные):

```json
{
  "summary": {
    "revenue": "250000.00",
    "cost_of_goods_sold": "150000.00",
    "gross_profit": "100000.00",
    "gross_margin_percent": "40.00",
    "other_expenses_total": "35000.00",
    "net_profit": "65000.00",
    "net_margin_percent": "26.00"
  }
}
```

Правила:

1. Поля обязательны всегда (нет расходов — `"0.00"`), чтобы карточки на фронте были стабильны.
2. Период/филиал применяются к прочим расходам так же, как к выручке (включительно, местное время — правила `docs/market/analytics.md`).
3. Желательно также поддержать ключ `other_expenses` в `GET /main/analytics/cards/details/` (постраничный список операций для модалки деталей) — фронт подключит его следующим шагом.

## 4. Чек-лист приёмки

Данные за период: выручка 10000, себестоимость 6000, закупка сырья на 3000 (расход кассы), аренда 1000 (расход кассы, approved), транспорт 500 (расход, pending).

| # | Проверка | Ожидание |
|---|---|---|
| 1 | `gross_profit` | 4000 (не изменился от нововведений) |
| 2 | `other_expenses_total` | 1000 (аренда; закупка исключена, pending не входит) |
| 3 | `net_profit` | 3000 |
| 4 | `net_margin_percent` | 30.00 |
| 5 | Период без прочих расходов | `other_expenses_total="0.00"`, `net_profit=gross_profit` |
| 6 | Выручка 0 | `net_margin_percent="0.00"`, не деление на ноль |
