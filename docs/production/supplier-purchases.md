# Производство/Магазин — Карточка поставщика: история закупок

**Страница:** `/crm/clients/:id` — карточка поставщика (фронт: `src/Components/Sectors/Market/ClientDetails/ClientDetails.jsx`, секция «История закупок»).
**Эндпоинт:** `GET /main/suppliers/{id}/purchases/` — **новый** (рядом с существующим `GET /main/suppliers/{id}/products/`).
**Статус:** фронт готов и ждёт эндпоинт; пока API отвечает 404, секция показывает заглушку «после обновления сервера».

---

## 1. Задача

Клиент (сектор Производство) хочет видеть в карточке поставщика **всю историю отношений**: сколько товара и когда у него закуплено, на какие суммы и как оплачено. Сейчас карточка показывает только текущие товары поставщика; истории закупок нет. Секция включена для секторов «Производство» и «Магазин».

## 2. Что считается закупкой

Каждое поступление товара/сырья от поставщика:

- в Производстве — закупка сырья (`RawMaterialAddModal`, `AddRawMaterials`) и закупка готового товара без рецепта с указанным поставщиком (`client` в `create-manual`);
- в Магазине — приход товара от поставщика (закупки склада).

Запись должна фиксироваться в момент операции со снимком цены/количества (последующее изменение товара историю не меняет).

## 3. Контракт API

```
GET /main/suppliers/{supplier_id}/purchases/?date_from=2026-06-01&date_to=2026-07-14&limit=50&offset=0
```

### Параметры (все опциональные)

| Параметр | Тип | Описание |
|---|---|---|
| `date_from`, `date_to` | date `YYYY-MM-DD` | Период, включительно, местное время (правила как в `docs/market/analytics.md`) |
| `limit`, `offset` | int | Пагинация; по умолчанию сортировка новые сверху |

### Ответ

```json
{
  "count": 42,
  "next": null,
  "previous": null,
  "summary": {
    "total_amount": "182500.00",
    "total_count": 42
  },
  "results": [
    {
      "id": "uuid",
      "product_name": "Мука в/с",
      "quantity": "500.000",
      "unit": "кг",
      "amount": "27500.00",
      "payment_type": "cash",
      "purchased_at": "2026-07-10T11:20:00"
    }
  ]
}
```

Правила:

1. `payment_type`: `cash` | `transfer` | `debt` | `prepayment` (фронт показывает «Оплачено» / «Долг» / «Предоплата»).
2. `amount` — сумма позиции закупки (цена × количество), Decimal-строка.
3. `summary` — по всему отфильтрованному периоду, не по странице.
4. `purchased_at` — ISO местное время компании.
5. Поставщик другой компании или не-поставщик — HTTP 404.
6. Пустая история — `results: []`, `summary` нули, HTTP 200.

## 4. Чек-лист приёмки

Данные: поставщик П1 с тремя закупками (10.06 — 1000 сом наличные; 05.07 — 2000 сом долг; 12.07 — 500 сом предоплата), поставщик П2 без закупок.

| # | Запрос | Ожидание |
|---|---|---|
| 1 | `GET /main/suppliers/П1/purchases/` | `count=3`, `summary.total_amount=3500`, новые сверху |
| 2 | `…?date_from=2026-07-01&date_to=2026-07-31` | 2 записи, `summary.total_amount=2500` |
| 3 | `GET /main/suppliers/П2/purchases/` | `results=[]`, нули, HTTP 200 |
| 4 | Закупка сырья в Производстве через модалку добавления сырья | появляется в истории с корректным `payment_type` |
| 5 | Запрос к поставщику чужой компании | HTTP 404 |
