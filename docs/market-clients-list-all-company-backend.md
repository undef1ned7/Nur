# Магазин: `GET /api/main/clients/` должен возвращать всех клиентов компании

## Проблема
На сфере **Магазин** эндпоинт `GET /api/main/clients/` сейчас возвращает **только клиентов,
созданных текущим пользователем** (кассиром/сотрудником). Из-за этого разные сотрудники видят
разные списки, а общий список клиентов компании недоступен.

## Требование
Возвращать **всех клиентов в рамках компании** пользователя, а не только созданных им.

- Фильтрация — по **компании** (мультиарендность), как в остальных списках
  (`company = request.user.company`), а не по `created_by = request.user`.
- Клиенты, созданные любым сотрудником компании (кассир, админ, владелец), должны быть видны
  всем сотрудникам этой компании (с учётом их прав доступа).
- Это касается и `GET /api/main/clients/` (список), и связанных выборок
  (`?type=client|suppliers|...`, `with-debts/` и т.п.) — фильтр по компании, не по автору.

## Что убрать/поправить на бэкенде
- В `get_queryset` списка клиентов убрать ограничение вида `filter(created_by=request.user)`
  (или `owner=request.user`) — оставить только `filter(company=request.user.company)`.
- Поле «кто создал» можно сохранять (`created_by`) для истории, но **не использовать** его
  для фильтрации выдачи.
- Права: если для роли нужно ограничение (например, агент видит только своих) — делать это
  **явно по роли**, а не для всех подряд.

## Совместимость
- Пагинация/поиск/сортировка/фильтры по типу — без изменений.
- Создание клиента (`POST /api/main/clients/`) — без изменений (клиент по-прежнему привязан к
  компании; `created_by` фиксируется).

## Чек-лист
- [ ] Список клиентов фильтруется по компании, а не по создателю.
- [ ] Клиент, созданный одним сотрудником, виден другим сотрудникам компании.
- [ ] `?type=...` и `with-debts/` тоже отдают всех клиентов компании.
- [ ] `created_by` сохраняется, но не влияет на выдачу.
