# Возврат продажи не уменьшает «Ожидаемую сумму» смены (Магазин)

## Проблема
На кассе Магазина при **возврате продажи** (`/crm/market/cashier`) поле **Ожидаемая сумма**
(`expected_cash`) смены не меняется. При закрытии смены расхождение считается неверно.

## Где смотрели (фронт — корректен)
- Фронт **не вычисляет** `expected_cash` — только показывает значение с бэкенда.
- Экран закрытия смены перед показом делает `GET` смены по id, т.е. берёт **актуальный**
  `expected_cash` сервера. Расхождение = `внесённая сумма − expected_cash`.
- Возврат уходит на бэкенд: `POST /main/pos/sales/{saleId}/return/`
  (для агента — `/main/agents/me/sales/{saleId}/return/`).

Вывод: правка нужна **на бэкенде** — обработчик возврата не корректирует кассу смены.

## Что сделать на бэкенде
При возврате продажи в рамках открытой смены (эндпоинт `.../sales/{saleId}/return/`):

1. Если оплата возвращаемой продажи была **наличными** — **уменьшить `expected_cash`** смены
   на сумму возврата (наличные физически уходят из кассы).
2. Симметрично продаже отразить возврат в агрегатах смены:
   - уменьшить `cash_sales_total` (или вести отдельный `returns_total` / `refund_total`,
     который вычитается из ожидаемой суммы);
   - при необходимости — создать движение по кассе типа «расход/возврат» на сумму возврата.
3. Для **безналичного** возврата (карта/перевод) `expected_cash` (наличная касса) **не менять** —
   влиять только на соответствующие безналичные показатели.
4. **Частичный возврат** (возврат части позиций) — уменьшать на фактическую сумму возвращённых
   позиций, а не всей продажи.
5. Возврат с признаком брака (`is_defect`) — уточнить, участвует ли в кассе (обычно деньги
   покупателю всё равно возвращаются наличными → `expected_cash` уменьшается).

## Формула (ориентир)
```
expected_cash = opening_cash
              + cash_sales_total        (продажи за наличные)
              + income_total            (внесения)
              − expense_total           (изъятия)
              − cash_returns_total       ← НУЖНО добавить учёт возвратов наличными
```

> **Возврат долговой продажи** должен также корректировать задолженность клиента (частичный и
> полный возврат) — вынесено в отдельный документ:
> `docs/market-sale-return-debt-backend.md`.

## Чек-лист
- [ ] Возврат наличной продажи уменьшает `expected_cash` смены.
- [ ] Частичный возврат уменьшает на сумму возвращённых позиций.
- [ ] Безналичный возврат не трогает `expected_cash`.
- [ ] Возврат отражён в агрегатах смены/движении по кассе (симметрично продаже).
- [ ] После возврата `GET` смены возвращает уже пересчитанный `expected_cash`.
