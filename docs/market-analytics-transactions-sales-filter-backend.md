# Магазин: список продаж по фильтру аналитики (`GET /main/pos/sales/`)

На странице `/crm/market/analytics` клик по блоку **«Транзакции»** открывает список продаж,
**отфильтрованный по текущему фильтру аналитики**. Фронт отправляет фильтр в **query-параметрах**
запроса `GET /main/pos/sales/`. Нужно, чтобы бэкенд их учитывал (сейчас, судя по коду, список
отдаётся без серверной фильтрации — фронт фильтрует как fallback на клиенте).

## Query-параметры (фильтр)
| Параметр | Описание |
|----------|----------|
| `date_from` | Начало периода `YYYY-MM-DD` (включительно) |
| `date_to` | Конец периода `YYYY-MM-DD` (включительно) |
| `payment_method` | `cash` \| `card` \| `transfer` \| `debt` (если задан) |
| `cashbox` | id кассы |
| `cashier` | id кассира |
| `min_total` | минимальная сумма чека |
| `max_total` | максимальная сумма чека |

- Все параметры **опциональны** и комбинируются логическим И.
- Ограничение по компании (мультиарендность) — как в остальных списках.
- Желательно поддержать пагинацию (`page`, `page_size`) — список может быть большим.

## Статусы/типы продаж (нужны фронту для подписи)
Фронт показывает бейдж по каждой продаже. Нужно, чтобы в объекте продажи были корректные
`status` и `payment_method`:

| Ситуация | Как определяем | Подпись на фронте |
|----------|----------------|-------------------|
| Оплачено | `status = paid` (и не долг) | **Оплачено** (зелёный) |
| **Отменено / возврат** | `status ∈ {canceled, cancelled, refunded, returned}` | **Отменено** (красный) |
| **Долг** | `payment_method = debt` или `status ∈ {debt, unpaid, credit}` | **Долг** (жёлтый) |
| Частичная оплата | `status ∈ {partial, partially_paid}` | **Частично** (жёлтый) |

Требования:
- В списке продаж возвращать **все** продажи периода, включая **отменённые** и **долговые**
  (не скрывать отменённые).
- `status` должен явно отражать отмену/возврат (`canceled`/`refunded`), чтобы отличать от
  оплаченных.
- Для долговых продаж — либо `payment_method = debt`, либо статус долга; желательно также отдавать
  остаток долга по продаже (напр. `debt_amount` / `balance_due`), чтобы можно было показать сумму
  долга.

## Поля в ответе (используются фронтом)
`id`, `status`, `payment_method`, `total`, `created_at` (или `paid_at`), опционально
`user_display` (кассир), `client`/`client_name`, `debt_amount`.

## Чек-лист
- [ ] `GET /main/pos/sales/` принимает `date_from`/`date_to` и фильтрует по ним.
- [ ] Учитывает `payment_method`, `cashbox`, `cashier`, `min_total`, `max_total`.
- [ ] В выдаче есть отменённые и долговые продажи с корректным `status`/`payment_method`.
- [ ] (Опц.) `debt_amount` по долговым, пагинация.
