# Аналитика склада: долги считаются отдельной графой (не расход) сфера Склад

Документ для фронта. Касается аналитики владельца по кассе в модуле **warehouse**.

Эндпоинты:
- `GET /api/warehouse/owner/analytics/` — аналитика владельца по своей компании.
- `GET /api/warehouse/owner/partners/<partner_company_id>/analytics/` — по компании-партнёру
  (та же структура ответа).

---

## Проблема (что было)

Кассовые операции с системной категорией **«Долги»** (`system_code=debt`) частично попадали
в обычный **расход/приход** кассы. В частности, погашение долга контрагентом или оплата долга
поставщику (документ с указанным контрагентом) уходили в графу «операции с контрагентами» либо
в разбивку расходов по категориям — и на UI выглядели как обычный расход. Из-за этого сальдо
кассы и цифры расхода были завышены.

## Что стало

Теперь **любой** кассовый документ с категорией «Долги» (с контрагентом или без) считается в
**отдельной графе «долг»** и:

- **не** входит в обычный приход/расход (`money_receipt_amount` / `money_expense_amount`);
- **не** входит в сальдо кассы (`money_net_amount`);
- **не** входит в графу «операции с контрагентами» (`money_counterparty_*`);
- **не** появляется в разбивке расходов/приходов по категориям (`money_expenses_by_category` /
  `money_receipts_by_category`);
- учитывается **только** в полях `money_debt_*`.

Контракт ответа **не изменился** — все поля `money_debt_*` уже существовали. Изменилось только
то, какие суммы в них попадают (теперь и долги с контрагентом тоже).

---

## Три независимые графы денег

Каждая кассовая операция попадает ровно в одну из трёх групп:

| Графа | Что это | Поля |
|-------|---------|------|
| **Обычная касса** | Приход/расход без контрагента и не «Долги». Формирует сальдо кассы. | `money_receipt_amount`, `money_expense_amount`, `money_net_amount` |
| **Долг** | Всё по категории «Долги» (`system_code=debt`), с контрагентом и без. | `money_debt_receipt_amount`, `money_debt_expense_amount`, `money_debt_net_amount` |
| **Операции с контрагентами** | Документы с контрагентом, кроме категории «Долги». Взаиморасчёты по сальдо. | `money_counterparty_receipt_amount`, `money_counterparty_expense_amount`, `money_counterparty_net_amount` |

Правило приоритета: **«Долги» важнее контрагента.** Если у документа стоит категория «Долги»,
он идёт в графу «долг», даже если контрагент указан.

`*_net_amount = *_receipt_amount − *_expense_amount` в каждой графе.

---

## Где встречаются эти поля в ответе

Одинаковый набор ключей повторяется на трёх уровнях:

### 1. `summary` — итоги за период
```json
{
  "summary": {
    "money_receipt_amount": "500.00",
    "money_expense_amount": "200.00",
    "money_net_amount": "300.00",

    "money_debt_receipt_amount": "30.00",
    "money_debt_expense_amount": "130.00",
    "money_debt_net_amount": "-100.00",

    "money_counterparty_receipt_amount": "0.00",
    "money_counterparty_expense_amount": "0.00",
    "money_counterparty_net_amount": "0.00"
  }
}
```

### 2. `details.cash_by_register[]` — разбивка по кассам
Те же ключи для каждой кассы: `money_receipt_amount`, `money_expense_amount`, `money_net_amount`,
`money_debt_receipt_amount`, `money_debt_expense_amount`, `money_debt_net_amount`,
`money_counterparty_*`.

### 3. `charts.money_by_date[]` — разбивка по датам
Те же ключи в каждой точке графика (плюс `date`, `docs_count`).

Кроме того, `details.money_expenses_by_category[]` и `details.money_receipts_by_category[]`
теперь содержат **только обычные** категории — строки «Долги» из них исключены.

---

## Что делать на фронте

- Если раньше вы показывали «Расход» как `money_expense_amount` — теперь эта цифра **чистая**
  (без долгов и без взаиморасчётов с контрагентами). Если нужно показать «сколько всего вышло из
  кассы, включая долги» — суммируйте графы явно:
  `money_expense_amount + money_debt_expense_amount (+ money_counterparty_expense_amount)`.
- Рекомендуется вывести «Долг» отдельной карточкой/строкой рядом с приходом/расходом, используя
  `money_debt_*`. Отрицательный `money_debt_net_amount` означает, что по долгам за период кассой
  выдано больше, чем получено.
- Разбивку расходов по категориям (`money_expenses_by_category`) можно показывать как есть — она
  больше не содержит служебную категорию «Долги».

---

## Как операция попадает в графу «долг»

Документ учитывается в `money_debt_*`, если у него системная категория платежа с
`system_code = "debt"` (в UI — «Долги»). Эта категория создаётся автоматически для каждой
компании/филиала; отдельно её заводить не нужно. Чтобы операция считалась долгом, при создании
денежного документа укажите `payment_category` = id категории «Долги».

Получить id категории: `GET /api/warehouse/money/categories/` — искать запись с
`system_code = "debt"`.

## Статус на фронте

Реализовано. Компоненты аналитики уже читают все поля и показывают долг отдельной графой:
- `src/Components/Sectors/Warehouse/Analytics/OwnerAnalyticsContent.jsx` — карточки
  «Погашение долга» / «Выплаты по долгу» / «Нетто по долгам», приход/расход с подписью
  «Без долгов и контрагентов», графики и таблицы с долговыми колонками.
- `src/Components/Sectors/Warehouse/Analytics/warehouseAnalyticsShared.js` — маппинг
  `money_debt_*` / `money_counterparty_*`.
Дополнительных изменений на фронте не требуется — цифры обновятся автоматически после правки бэка.
