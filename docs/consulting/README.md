# Консалтинг — доработки (спецификации для бэкенда)

Дата: 2026-07-23. Сектор «Консалтинг» (ключ сектора `consulting`, backend-домен
`/api/consalting/`). Фронт готов по всем пунктам ниже и работает с graceful-
fallback: пока эндпоинт не реализован (ответ `404/501`), экран показывает
понятную заглушку «раздел ещё не подключён», не ломая интерфейс.

Все денежные поля — `Decimal(12,2)`, проценты — `Decimal(5,2)` (0.00–100.00).
Все сущности мультитенантны (принадлежат компании), время — местное компании
(Asia/Bishkek, UTC+6), как в [docs/market/analytics.md](../market/analytics.md).

## Порядок внедрения (рекомендуемый)

1. **#2 role_prices** — база для корректных сумм в продажах, воронке, аналитике,
   зарплате. Делать первым.
2. **#5 Зарплата** и **#1 Аналитика** зависят от корректной суммы сделки (#2).
3. **#4 Воронка** — гарантии серверных сайд-эффектов при завершении лида
   (создание сделки/абонентки/начисления зарплаты).
4. **#6 Матрица** — читает абонентские платежи, создаваемые при завершении лида.
5. **#3 Лиды/WhatsApp** — независимый модуль, можно параллельно.
