# Консалтинг — Абонентская матрица клиентов

**Страница:** `/crm/consulting/client` → вкладка **«Абонентская матрица»**
(фронт: `src/Components/Sectors/Consulting/client/SubscriptionMatrix.jsx`,
API-слой `src/api/consultingSubscriptions.js`).
**Статус:** ⚠️ Бэкенд не реализован. Фронт готов и показывает заглушку при
`404/501`.

## 1. Задача

Excel-подобная длинная таблица:

- **Строки** = «клиент × купленная услуга». Один клиент с несколькими услугами
  даёт **несколько строк** (одно ФИО повторяется).
- **Колонки** = месяцы выбранного периода (календарь).
- **Ячейка** = абонентский платёж клиента за эту услугу в этом месяце и его
  статус (оплачено / запланировано / просрочено / нет платежа).
- Липкие левые колонки: ФИО, услуга, размер абонплаты. Горизонтальный скролл по
  месяцам. Внизу строка «Итого оплачено» по каждому месяцу.

## 2. Эндпоинт

```
GET /consalting/subscription-matrix/?month_from=YYYY-MM&month_to=YYYY-MM&search=…
```

- `month_from`, `month_to` — включительные границы (месяцы). По умолчанию фронт
  просит последние 6 месяцев.
- `search` — по ФИО клиента или названию услуги (необязательный).
- Права: owner/admin — все клиенты; сотрудник — только «свои» (где он продавец/
  владелец лида), если так принято в проекте.

### Ответ

```jsonc
{
  "months": ["2026-02", "2026-03", "2026-04", "2026-05", "2026-06", "2026-07"],
  "rows": [
    {
      "client_id": "cli-1",
      "client_name": "Иванов Иван",
      "service_id": "svc-1",
      "service_name": "Внедрение CRM",
      "subscription_amount": 5000,
      "subscription_period": "month",         // month|year
      "cells": {
        "2026-02": { "amount": 5000, "status": "paid" },
        "2026-03": { "amount": 5000, "status": "paid" },
        "2026-04": { "amount": 5000, "status": "overdue" },
        "2026-05": { "amount": 5000, "status": "planned" }
        // отсутствующий месяц = нет платежа (в UI «—»)
      }
    },
    {
      "client_id": "cli-1",                   // тот же клиент, другая услуга → отдельная строка
      "client_name": "Иванов Иван",
      "service_id": "svc-2",
      "service_name": "Маркетинг",
      "subscription_amount": 3000,
      "subscription_period": "month",
      "cells": { "2026-05": { "amount": 3000, "status": "paid" } }
    }
  ]
}
```

Статусы ячейки: `paid` (оплачено), `planned` (запланировано, срок не наступил),
`overdue` (просрочено). Если ключа `status` нет, но `paid: true` — считается
оплаченным (фронт это учитывает).

## 3. Источник данных

Абонентские платежи создаются при завершении лида/продажи с абонентским тарифом
(см. [funnel-crm-logic.md](./funnel-crm-logic.md) §3.2). Каждый период абонентки
= строка расписания (`installment`) со статусом и суммой — та же модель, что
питает карточку клиента (`getClientSubscriptionSchedule`,
`ConsultingClientDetail.jsx`). Матрица — это сводка тех же расписаний по всем
клиентам, разложенная по месяцам.

Правила:

- Годовой тариф (`subscription_period="year"`) — платёж раз в год; в матрице
  показывается в месяце списания (остальные месяцы года — пусто).
- `overdue` = срок платежа прошёл, а оплата не проведена.
- Строка есть, только если у клиента реально есть абонентская услуга (не для
  всех клиентов подряд).

## 4. Производительность

Таблица может быть большой (клиенты × услуги × месяцы). Рекомендуется:

- Ограничить период (фронт по умолчанию 6 мес.).
- При необходимости — пагинация строк (`page`, `page_size`) с сохранением
  формата `{ months, rows, count }`; фронт легко расширить.
