# API запросов на аванс (для кассы) — Backend-спецификация

Отдельный API для оператора кассы: просмотр списка заявок на аванс по зарплате и их одобрение/отклонение. При одобрении сумма сразу снимается с основной суммы к выплате по строке начисления.

Связанная спецификация: [building_payroll_backend.md](building_payroll_backend.md) (разделы 5.2, 6).

---

## Базовые данные

**Префикс (на усмотрение backend):**
- `GET/POST /api/building/salary/advance-requests/...`  
  или  
- `GET/POST /api/building/cash-register/advance-requests/...`

**Права:** пользователи с правом `can_view_building_cash_register` (или `owner`/`admin`/`superuser`).

**Контекст:** заявка на аванс создаётся в модуле зарплаты (корректировка типа `advance` со статусом `pending`). Здесь обрабатываются только просмотр списка и действия одобрить/отклонить.

---

## 1. Список запросов на аванс

- **`GET /salary/advance-requests/`**  
  или **`GET /cash-register/advance-requests/`**

Возвращает заявки на аванс со статусом **`pending`** (ожидают одобрения кассы).

**Фильтры (query-параметры, опционально):**
- `cashbox` — uuid кассы
- `shift` — uuid смены
- `residential_complex` — uuid ЖК
- `payroll` — uuid периода начислений
- `status` — при необходимости только `pending` по умолчанию

**Поля в элементах списка (пример):**
- `id` — UUID заявки (или ID корректировки-аванса)
- `payroll_line_id`, `payroll_line` — строка начисления (или вложенный объект)
- `employee`, `employee_display` — сотрудник
- `amount` — сумма аванса
- `cashbox`, `cashbox_display` — касса из заявки
- `shift`, `shift_display` — смена (если указана)
- `created_at`, `requested_at` — когда создана заявка
- `status` — `pending` (в списке только такие)
- при необходимости: `residential_complex`, `payroll`, `period` — для фильтрации и отображения

Пагинация — по стандарту (например, `PageNumberPagination`).

## 2. Одобрить заявку на аванс

- **`POST /salary/advance-requests/{id}/approve/`**  
  или **`POST /cash-register/advance-requests/{id}/approve/`**

**Тело (опционально):**

```json
{
  "paid_at": "2026-03-10T14:00:00+06:00"
}
```

**Поведение backend:**
1. Проверить, что заявка существует и в статусе `pending`.
2. Проверить, что остаток по строке начисления позволяет выплатить аванс: `amount <= (net_to_pay - paid_total)`.
3. Пометить корректировку-аванс как проведённую (`status=completed` или аналог).
4. **Снять сумму с основной суммы к выплате:** увеличить `paid_total` по строке начисления на `amount` аванса.
5. Создать в кассе запись расхода (CashFlow: тип `expense`, статус `approved`, привязка к данной операции аванса через `source_business_operation_id`).
6. Вернуть успешный ответ (например, объект заявки со статусом `approved`/`completed`).

При ошибке (недостаточно остатка по строке, касса отклонила создание расхода и т.п.) вернуть `400`/`409` с текстом; сумму по строке не менять.

---

## 3. Отклонить заявку на аванс

- **`POST /salary/advance-requests/{id}/reject/`**  
  или **`POST /cash-register/advance-requests/{id}/reject/`**

**Тело (опционально):**

```json
{
  "reason": "Недостаточно средств в кассе"
}
```

**Поведение backend:**
1. Проверить, что заявка существует и в статусе `pending`.
2. Пометить корректировку-аванс как отклонённую (`status=rejected`). Сумму по строке **не** менять — `paid_total` не увеличивается, остаток к выплате остаётся прежним.
3. Сохранить причину отклонения в записи (поле `reject_reason` или в `comment`), если передано.
4. Вернуть успешный ответ.

**Итог:** при одобрении деньги сразу снимаются с основной суммы к выплате; при отклонении сумма к выплате не меняется.
