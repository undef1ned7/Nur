# Магазин: фильтр POS-продаж по клиенту (`GET /main/pos/sales/?client=<id>`)

Касается карточки клиента `/crm/clients/{id}` — там появился блок **«История продаж»**,
показывающий POS-продажи этого клиента.

## Что отправляет фронт

`GET /api/main/pos/sales/?client=<client_id>&page=1&limit=100`

| Параметр | Описание |
|----------|----------|
| `client` | UUID клиента — вернуть **только** продажи этого клиента |
| `page`, `limit` | Пагинация (как сейчас) |

## Текущее поведение / что нужно

Сейчас список продаж, судя по всему, не поддерживает фильтр `client` — фронт вынужден
грузить страницу продаж и фильтровать по `sale.client === id` на клиенте. Это ненадёжно:
если продаж больше `limit`, часть истории клиента теряется.

**Нужно:** серверная фильтрация по `client`:
- вернуть все продажи, у которых `client == <id>` (учитывая пагинацию);
- совместимо с существующими `search`, `page`, `limit` (и, при наличии, `date_from`/`date_to`);
- неизвестное значение → пустой список, не 500;
- ограничение по компании как обычно.

В объекте продажи поле `client` уже приходит (uuid или null) — контракт ответа менять
не нужно.

## Фолбэк на фронте

До готовности бэка фронт дофильтровывает полученный список по `client` на своей стороне
(это уже реализовано), поэтому параметр можно внедрять без синхронного релиза.

## Чек-лист
- [ ] `GET /main/pos/sales/` принимает `client=<uuid>` и фильтрует выдачу.
- [ ] Работает вместе с `search`/`page`/`limit`.
- [ ] Продажи других клиентов и без клиента в выдачу не попадают.
