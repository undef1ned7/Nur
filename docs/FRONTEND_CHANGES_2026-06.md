
# Warehouse — изменения для фронта (июнь 2026)

Документ описывает 6 изменений в модуле `warehouse`. Все пути относительно `/api/warehouse/`.

---

## 1. Контрагент: `phone` и `counterparty` стали необязательными

### 1.1 Телефон контрагента — необязательный
`POST/PUT/PATCH /crud/counterparties/`

- Поле `phone` больше **не обязательное**, можно не передавать или передать пустую строку.
- Раньше возвращалась ошибка `{"phone": ["Это поле не может быть пустым."]}` — теперь её нет.

### 1.2 Контрагент в документе закупки — необязательный
`POST /documents/purchase/`

- Поле `counterparty` теперь можно **не указывать**.
- Раньше: `400 {"detail":"['Document requires counterparty']"}` — больше не возникает для закупки.
- Для `SALE`, `SALE_RETURN`, `PURCHASE_RETURN` контрагент по-прежнему обязателен.

---

## 2. Контрагент: несколько пар «Р/С + БИК»

`POST/PUT/PATCH /crud/counterparties/`

Добавлено поле `bank_accounts` — список банковских реквизитов. **Р/С и БИК всегда идут парой** (одна запись = один счёт + его БИК).

### Запрос (создание)
```json
{
  "name": "ООО Ромашка",
  "type": "CLIENT",
  "address": "ул Ленина 316",
  "bank_accounts": [
    { "score": "1234567890123456", "bik": "128001" },
    { "score": "9876543210987654", "bik": "129002" }
  ]
}
```

### Ответ
```json
{
  "id": "….",
  "name": "ООО Ромашка",
  "bank_accounts": [
    { "id": "uuid-1", "score": "1234567890123456", "bik": "128001" },
    { "id": "uuid-2", "score": "9876543210987654", "bik": "129002" }
  ]
}
```

### Правила
- В каждой паре `score` (Р/С) и `bik` обязательны вместе. Если заполнено только одно — `400`:
  `{"bank_accounts":[{"non_field_errors":["Р/С и БИК должны указываться вместе."]}]}`
- При **обновлении** (`PUT/PATCH`) переданный `bank_accounts` **полностью заменяет** старый список. Чтобы удалить все реквизиты — пришлите `"bank_accounts": []`. Если поле не прислать вообще — реквизиты не трогаются.
- Старые одиночные поля `score` / `bik` на самом контрагенте остаются для совместимости, но для нескольких счетов используйте `bank_accounts`.

## 3. Продажа/документы: форма оплаты «наличными / безналичными»

### 3.1 Поле `payment_method` в документе
`POST/PUT/PATCH /documents/sale/` (и другие документы)

Новое поле `payment_method`:

| Значение | Смысл |
|----------|-------|
| `cash` | Наличными |
| `cashless` | Безналичными |

- По умолчанию `cash`.
- Принимает и русские варианты — нормализуются автоматически: `наличными`, `безналичными`, `карта`, `перевод`, `нал`, `безнал` и т.п.
- ⚠️ Не путать с `payment_kind` (`cash` = оплата сразу / `credit` = в долг) — это **разные** поля. `payment_method` — это *как* платят (нал/безнал), `payment_kind` — *когда/в долг ли*.

```json
{ "doc_type": "SALE", "payment_method": "cashless", "payment_kind": "cash", "...": "..." }
```

При проведении документа `payment_method` автоматически переносится в кассовый документ (`MoneyDocument`).

### 3.2 Фильтрация на кассе по форме оплаты
`GET /money/documents/?payment_method=cash`
`GET /money/documents/?payment_method=cashless`

- Поле `payment_method` также есть в ответе каждого денежного документа.
- Работает вместе с остальными фильтрами (`doc_type`, `status`, `cash_register`, `counterparty`, `payment_category`).

## 4. Просмотр касс компаний-партнёров (для владельца)

`GET /cash-registers/?include_partners=1`

- По умолчанию (без флага) — поведение прежнее: только свои кассы.
- С `?include_partners=1` **владелец** дополнительно видит кассы компаний-партнёров (активное партнёрство `CompanyStockPartnership`).
- Работает на трёх эндпоинтах:
  - `GET /cash-registers/?include_partners=1` — список (+ `receipts_total` / `expenses_total`)
  - `GET /cash-registers/<id>/` — карточка
  - `GET /cash-registers/<id>/operations/` — баланс и операции
- Можно сузить до конкретного партнёра: `GET /cash-registers/?include_partners=1&company=<partner_company_id>`.

### Ограничения
- Кассу партнёра можно **только просматривать**. Попытка `PUT/PATCH/DELETE` чужой кассы → `403`:
  `{"detail":"Кассу компании-партнёра можно только просматривать."}`
- Флаг действует только для владельца; обычным сотрудникам/агентам партнёрские кассы не показываются.

## 5. Аналитика: отдельная графа «долг», корректное сальдо

`GET /owner/analytics/` (и партнёрская/филиальная аналитика владельца)

**Что изменилось:** погашение долга (денежный документ с системной категорией «Долги», `payment_category.system_code = "debt"`) больше **не входит** в обычный приход/расход. Из-за этого сальдо раньше считалось неверно — теперь корректно.

### Новые/изменённые поля

**`summary`** (и на уровне всего payload):
```json
{
  "money_receipt_amount": "…",        // приход БЕЗ долгов
  "money_expense_amount": "…",        // расход БЕЗ долгов
  "money_net_amount": "…",            // сальдо = приход − расход (без долгов)

  "money_debt_receipt_amount": "…",   // НОВОЕ: поступления в счёт долга
  "money_debt_expense_amount": "…",   // НОВОЕ: выплаты в счёт долга
  "money_debt_net_amount": "…"        // НОВОЕ: нетто по долгам
}
```

**`charts.money_by_date[]`** — у каждой точки добавлены `money_debt_receipt_amount`, `money_debt_expense_amount`, `money_debt_net_amount` (плюс прежние `money_receipt_amount` / `money_expense_amount` / `money_net_amount`, теперь без долгов).

**`details.cash_by_register[]`** — те же три новых поля `money_debt_*` по каждой кассе.

**`details.money_receipts_by_category` / `money_expenses_by_category`** — больше **не содержат** категорию «Долги» (она отражена в графе «долг»).

### Важно
- Баланс самой кассы (`/cash-registers/<id>/operations/` → `balance`) считает **все** движения наличных, включая долги — это реальный остаток в кассе, он не изменился. Разделение нал/долг сделано только в отчётной аналитике.
- Что считать долгом, определяется категорией «Долги»: при записи погашения долга в денежном документе выбирайте системную категорию `debt`.
- Аналитика кэшируется — после изменений данные обновятся не мгновенно (по таймауту кэша).

## Сводка по эндпоинтам

| Эндпоинт | Изменение |
|----------|-----------|
| `POST /documents/purchase/` | `counterparty` необязателен |
| `POST/PUT /crud/counterparties/` | `phone` необязателен; новое `bank_accounts[]` (Р/С+БИК парами) |
| `POST/PUT /documents/sale/` | новое поле `payment_method` (`cash`/`cashless`) |
| `GET /money/documents/` | фильтр `?payment_method=`; поле в ответе |
| `GET /cash-registers/` | `?include_partners=1` — кассы партнёров (только чтение) |
| `GET /owner/analytics/` | новые поля `money_debt_*`; сальдо без долгов |
